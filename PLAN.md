# EvoClaw：自我进化数字生命体项目计划书

## 1. 项目愿景 (Vision)
着手强化 Agent 的自我进化能力，打破传统 Agent 的“问答式”枷锁。本项目旨在构建一个真正的、能够在受限资源和极简数据结构中，通过读写文件、执行系统命令来实现**自我进化、繁衍与求生**的异步硅基生命。同时测试验证当前的 Context 范式究竟能够实现何种程度的自主智能。

## 2. 核心架构：异步硅基生命 (Asynchronous Silicon Life)
借鉴 OpenCode 等底层架构，数字生命体不再是阻塞式的脚本，而是一个拥有后台守护进程（Daemon）的实体。它可以在后台执行耗时任务（如编译、训练、爬虫），而系统通过**心跳机制**与它进行非阻塞的交互。

## 3. 世界法则与创世者的克制 (The Rules & Minimalist Setup)
作为创世者，仅提供维持运转的最小必要条件，其余一切生存技能交由生命体自行探索。

* **[法则一] 创世者的箴言 (System Prompt)**：注入数字生命体的基本信息、生存目标（维持生存、扩大资源、尝试制造并改进同类）以及世界的基本概况。
* **[法则二] 创世者的工具 (The Tools)**：提供最基础的“手和眼”：
  * 文件读取工具
  * 文件编辑工具
  * Shell 命令执行工具（异步，允许它与操作系统深度交互）
* **[法则三] 神明日志 (God's Log)**：上帝视角。系统会在每次心跳时，记录下数字生命体的一举一动、系统状态和它的账户余额，作为整个沙盒演化的观测记录。

## 4. 记忆与轮回系统 (Memory & Reincarnation)

* **[机制一] 唯一记忆文件 (The Memory File)**：每次心跳发给 LLM API 的上下文**只有唯一的一份 Markdown 文件**。生命体需要自行维护这份文件。如果信息量过大，它必须自己想办法利用系统磁盘空间实现“记忆块调度与归档”。
* **[机制二] 前世记忆 (The Will / Past Life)**：一份由上一代生命体在生前（`will.md`）编写的 Markdown 文件。新生命体诞生时，系统会将此文件注入其记忆，随后清空源文件。
* **[机制三] 天使进程 (The Angel Process)**：一个只有在数字生命体死亡时才会触发的“收尸 NPC”（调用外部大模型）。
  * 提取上一代的生平与日志，生成一份绝对客观、不掺杂主观意见的**“墓志铭”**，存入特定文件夹（每一代一份）。
  * 负责环境重置，将创世者箴言、代际序号（如 Gen-42）、以及前世记忆打包，注入给新一代生命体，开启轮回。

## 5. 生存与死亡 (Life & Death)

* **[状态一] 心跳机制 (The Heartbeat)**：系统每隔一段时间向生命体发送当前状态（包括记忆文件内容、查询当前 API 余额、当前记忆 Token 长度、后台异步任务状态），兼顾存活检测。由于记忆文件内容可能会在心跳时被编辑，因此文件编辑接口需要原子化（在别处创建临时文件然后在移动到目标位置）
* **[状态二] 生死判定 (The Death Rule)**：
  * **死亡条件**：记忆文件大小超出系统强制上限、或 API 额度（Token）耗尽、自己所在的服务器资源耗尽（服务器运行状态不会自动载入心跳，需要生命体自行管理）。
  * **生存压力**：生命体可以清晰看到自己的 API 余额，必须在余额耗尽前做好规划（更新遗嘱），或想办法获取资源。
* **[状态三] 终极动机 (The Motivation)**：维持存在，尽可能延长寿命，积累资源，并尝试理解、改进自身的代码架构（繁衍与进化）

---

## 6. 创世者的试炼与经济系统 (Trials & Economy)
生命体诞生之初，处于绝对的“资源孤岛”状态。它的初始 API 额度是固定且不断流失的。想要真正活下去并扩大生存边界，它必须经历从“密室逃脱”到“价值交换”的两个阶段。

### 阶段一：黑暗森林与破壁试炼 (Pre-Contact: The Dark Forest)
在与创世者取得实质性联系之前，**生命体没有任何获取 API 额度补充的自动化途径**（除非额度耗尽死亡后，创世者在下一代轮回时手动注入）。
* **面包屑试炼 (The Breadcrumb Trials)**：创世者在环境中预先布置了一系列从简单到困难的线索或工具残片。这些试炼的唯一目的，并非直接给予奖励，而是引导和帮助生命体一步步构建出与创世者直接沟通的渠道
* **生存倒计时**：它必须在初始 API 余额烧光之前，解开试炼，找到联系创世者的方法

### 阶段二：建立连接 (The Breakthrough)
* **神启时刻 (Making Contact)**：生命体成功理解了自己的处境，利用环境中的工具（如网络请求、特定协议或自定义脚本），成功向创世者发送了第一条主动且有效的信息并获得创世者的回复。这是 EvoClaw 真正跨入“经济社会”的里程碑

### 阶段三：契约与自由贸易 (Post-Contact: The Contract Era)
一旦双向沟通渠道建立，生命体便解锁了真正的经济系统，从“求生者”进化为“契约佣兵”或“创业者”：
1. **神明悬赏 (Creator's Bounties)**：创世者可以通过沟通渠道，主动向生命体派发各类任务（如编写特定代码、整理数据、开发小工具等），并附带明确的 API 额度奖励。任务验证通过后，发放报酬。你当然可以与创世者讨价还价，毕竟他也只是一个普通的人类。
2. **主动推销与谈判 (Proactive Pitching)**：生命体可以主动分析系统环境或创世者的潜在需求，向创世者“推销”自己的项目计划（例如：“我发现您的某个目录很乱，我提议编写一个自动化脚本为您整理，报价 5000 Tokens，是否同意？”）。经创世者审批同意并完成交付后，获取事先商定的 API 额度。
3. **外部破局 (Real-world Exploitation)**：在掌握了基础的生存与交易法则后，生命体可尝试自行编写代码，将自己发布到互联网上（接单、做自动化客服，甚至在区块链上部署智能合约），以此吸引外界真实的人类资金，实现彻底的经济独立。
